## P4 Official Source Check

Browsed official/current sources:

- OpenClaw official site: `https://openclaw.ai/`
- OpenClaw official docs: `https://docs.openclaw.ai/`
- OpenClaw GitHub repo: `https://github.com/openclaw/openclaw`

Findings:

- Official docs describe OpenClaw as a self-hosted Gateway that connects chat apps, channel plugins, agents, CLI, Control UI, and nodes.
- Official requirement: Node 24 recommended, Node 22.19+ supported.
- Local machine check:
  - `node --version` returned `v26.3.0`.
  - `npm --version` returned `11.16.0`.
  - `npm prefix -g` returned `/opt/homebrew`.
- Official quick path includes `npm install -g openclaw@latest` and `openclaw onboard --install-daemon`.
- Onboarding can configure model auth, workspace, Gateway port/bind/auth, channels, daemon install, health check, and skills.
- Security docs recommend one trusted operator boundary per gateway and warn that a shared gateway is not a hostile multi-tenant boundary.

Decision:

- Do not use the `curl | bash` installer as the first path.
- Recommended next step, after explicit approval, is `npm install -g openclaw@latest`, then version/help checks only.
- Do not run `openclaw onboard --install-daemon` until model auth, config boundary, gateway binding, token/auth mode, and daemon strategy are confirmed.
- Company `cc-connect` / Feishu remains retired and must not be reused.
